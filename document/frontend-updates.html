Frontend Customization

Frontend Customization Guide: Learn how to customize the SchoolHub Platform frontend, including branding, styling, components, pages, and configuration options.

Frontend Structure

The frontend is built with React.js using Material-UI components and Redux for state management. All code is located in the client/src/ directory:

Feature Configuration

Update User Roles

User roles are configured in the backend models:

1. User Model (Backend)

Update roles in server/models/userModel.js:

role: {
  type: String,
  required: true,
  enum: ["teacher", "student", "admin"],
  index: true,
}

2. Frontend Role Checks

Role-based UI rendering in React components:

// Example: Role-based component rendering
import { useSelector } from 'react-redux';

const Dashboard = () => {
  const { userInfo } = useSelector((state) => state.user);
  
  return (
    <>
      {userInfo.role === 'admin' && }
      {userInfo.role === 'teacher' && }
      {userInfo.role === 'student' && }
    
  );
};

// Example: Protected route component
const ProtectedRoute = ({ children, allowedRoles }) => {
  const { userInfo, isAuth } = useSelector((state) => state.user);
  
  if (!isAuth) {
    return ;
  }
  
  if (!allowedRoles.includes(userInfo.role)) {
    return ;
  }
  
  return children;
};

API Integration

SchoolHub Platform uses Axios or Fetch API for backend communication. API utilities are in client/src/utils/api.js:

  • API Base URL: Configured via REACT_APP_API_URL environment variable
  • Authentication: JWT tokens sent in Authorization header
  • Error Handling: Centralized error handling in API utility functions
  • Request Interceptors: Automatically add auth tokens to requests

Build and Deployment

Development

Run frontend and backend separately:

# Start backend server (from server/ directory)
cd server
npm start
# Backend runs on http://localhost:5000

# Start frontend (from client/ directory, in new terminal)
cd client
npm start
# Frontend runs on http://localhost:3000

Production Build

# Build frontend (from client/ directory)
cd client
npm run build

# Build creates optimized production files in client/build/
# Serve the build folder with a static file server or deploy to hosting service

# Backend production (from server/ directory)
cd server
npm start
# Or use PM2: pm2 start npm --name "schoolhub" -- start
Note: This is a MERN stack application. Frontend (React) and backend (Node.js/Express) are separate applications. Frontend makes API calls to the backend server.

Environment Variables

Create .env files in respective directories:

# Backend Environment Variables (server/.env)
MONGODB_URI=mongodb://localhost:27017/schoolhub
JWT_SECRET=your_jwt_secret_here
PORT=5000

# Frontend Environment Variables (client/.env)
REACT_APP_API_URL=http://localhost:5000/api
REACT_APP_SOCKET_URL=http://localhost:5000

# See env-setup.html for the complete variable list

Troubleshooting

Common Issues:
  • API Connection Failed: Check REACT_APP_API_URL is set correctly in client/.env
  • Backend Not Responding: Verify backend server is running and accessible
  • Build Errors: Check for JavaScript errors and ensure all dependencies are installed
  • Styling Issues: Ensure Material-UI theme is properly configured
  • Authentication Fails: Verify JWT_SECRET matches between frontend and backend
  • CORS Errors: Check backend CORS configuration allows frontend origin
  • File Upload Issues: Check file upload middleware and storage configuration
