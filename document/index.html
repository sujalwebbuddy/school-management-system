System Architecture

Enterprise Tech Stack:
  • Frontend: React.js with Material-UI, Syncfusion, and Bootstrap components
  • Backend: Node.js with Express.js and comprehensive middleware architecture
  • Database: MongoDB with Mongoose ODM and multi-tenant data isolation
  • Authentication: JWT with bcrypt, role-based access control, and secure approval workflows
  • AI/ML: TensorFlow.js with COCO-SSD for person detection and intelligent features
  • State Management: Redux Toolkit with advanced async handling
  • File Storage: AWS S3 with pre-signed URLs and automatic upload processing
  • Real-time Communication: Socket.io with advanced chat and notification systems
  • Payment Processing: Stripe integration with webhook handling and subscription management
  • UI Components: ApexCharts, Syncfusion grids, calendars, and rich text editors
  • Email Services: Nodemailer for automated notifications and communications
  • Deployment: Vercel (frontend) and AWS EC2/VPS (backend) with Docker support

User Roles & Permissions

User Roles:

SchoolHub Platform supports three distinct user roles with different permissions and access levels:

  • Administrator: Full system access - manage users, classes, subjects, approve registrations, system configuration
  • Teacher: Class management, homework assignment, student grading, communication with students and admin
  • Student: View assignments, submit homework, participate in class discussions, communicate with teachers

Note: All user registrations require admin approval before accounts become active. This ensures secure and controlled access to the platform.

Prerequisites

Before you begin, ensure you have:
  • Node.js 16+ installed on your system
  • MongoDB database (local installation or MongoDB Atlas cloud)
  • Basic knowledge of JavaScript/React for customization
  • Web server or hosting platform (Vercel, AWS EC2, or similar)
  • Git for version control (optional but recommended)

Quick Start

  1. Clone: Get the SchoolHub repository from GitHub
  2. Install Backend: Navigate to server/ and run npm install
  3. Install Frontend: Navigate to client/ and run npm install
  4. Setup Database: Configure MongoDB connection in environment variables
  5. Environment Config: Set up all required environment variables
  6. Start Backend: Run npm start in server directory
  7. Start Frontend: Run npm start in client directory
  8. Create Admin: Use the admin creation script to set up your first admin account
Pro Tip: Start with the local development setup first. Once everything is working locally, follow our deployment guides for production hosting.
