User Roles & Permissions

Role-Based Access Control: SchoolHub Platform implements comprehensive role-based permissions to ensure secure and appropriate access to features. Each user role has specific capabilities and restrictions designed for educational workflows.

User Roles Overview

Role Hierarchy

SchoolHub Platform supports three distinct user roles with escalating permissions:

Administrator (Highest Level)

  • Full System Access: Complete control over all platform features
  • User Management: Approve/reject user registrations, manage all accounts
  • Class Management: Create/edit/delete classes, assign students and teachers
  • Subject Management: Create subjects, assign teachers to subjects
  • System Configuration: Modify system settings and configurations
  • Reports & Analytics: Access to all system reports and analytics
  • Security Controls: Manage permissions and access controls

Teacher (Mid Level)

  • Subject Management: Manage assigned subjects and related content
  • Homework Creation: Create, edit, and delete homework assignments
  • Student Assessment: Grade submissions, provide feedback
  • Class Communication: Send messages to students and administrators
  • Progress Tracking: Monitor student performance in assigned subjects
  • Resource Sharing: Upload and share educational materials
  • Limited User Management: View students in assigned classes

Student (Base Level)

  • Assignment Access: View and download homework assignments
  • Submission Management: Submit completed work with file attachments
  • Grade Viewing: Access grades and teacher feedback
  • Class Communication: Send/receive messages from teachers
  • Profile Management: Update personal information and avatar
  • Resource Access: Download shared educational materials
  • Progress Tracking: View personal academic progress

  • Data Isolation: Complete separation of data between organizations
  • Feature Gating: Subscription-based feature availability

Security Best Practices

User Authentication

  • JWT Tokens: Secure token-based authentication with expiration
  • Password Hashing: Bcrypt encryption for all passwords
  • Role Validation: Server-side role verification on all requests
  • Session Management: Automatic logout on token expiration

Data Protection

  • Input Validation: All user inputs validated and sanitized
  • File Upload Security: Restricted file types and size limits
  • Database Security: Proper indexing and query optimization
  • API Rate Limiting: Protection against abuse and DDoS attacks

Role-Based Route Protection

Routes are protected based on user roles using middleware:

// Middleware example (server/middlewares/authMiddleware.js)
const authMiddleware = async (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) {
    return res.status(401).json({ msg: 'No token provided' });
  }
  
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.userId = decoded.sub;
  req.userRole = decoded.role;
  next();
};

// Role-based access control
const adminOnly = (req, res, next) => {
  if (req.userRole !== 'admin') {
    return res.status(403).json({ msg: 'Admin access required' });
  }
  next();
};

Common Permission Issues

Permission Issues:
  • Access Denied Errors: Verify user role is correctly set in database and JWT token
  • Features Not Showing: Check frontend role checks match user's actual role
  • Unauthorized Actions: Ensure backend middleware validates roles before processing requests
  • Role Not Updating: Verify admin has permission to change user roles
  • Token Expiration: Check JWT token expiration and refresh mechanism

üìù Best Practices

Security Best Practices:
  • Always validate permissions on the backend, never rely solely on frontend checks
  • Use middleware to protect routes based on user roles
  • Implement proper JWT token validation for all authenticated requests
  • Log all permission-related actions for audit purposes
  • Regularly review and update role permissions as needed
  • Test role-based access control thoroughly before deployment
  • Keep role definitions consistent between frontend and backend
  • Document all role permissions and access levels clearly
