Push Code to GitHub

Overview

This guide will walk you through downloading the code from CodeCanyon, setting up a GitHub repository, and pushing your code to GitHub. This is a prerequisite step before deploying to Vercel or AWS EC2.

Requirements

  • CodeCanyon purchase confirmation
  • GitHub account (free)
  • Git installed on your computer
  • Code editor (VS Code recommended)

Step 1: Download from CodeCanyon

  1. Log in to CodeCanyon and go to "Downloads"
  2. Download "SchoolHub Platform" ZIP file
  3. Extract the ZIP file to a folder

Step 2: Set Up Git

# Install Git from: https://git-scm.com/downloads
# Configure Git (first time only)
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Step 3: Create GitHub Repository

  1. Go to github.com and sign in
  2. Click "+" → "New repository"
  3. Name: "schoolhub-platform", set to Private, click "Create repository"
  4. Copy the repository URL (e.g., https://github.com/yourusername/schoolhub-platform.git)

Step 4: Push to GitHub (Command Line)

# Navigate to your project folder
cd /path/to/schoolhub-platform

# Initialize Git and push to GitHub
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/schoolhub-platform.git
git push -u origin main

Step 5: Alternative - GitHub Desktop (Optional)

  1. Download and install GitHub Desktop
  2. Sign in to your GitHub account
  3. Click "Clone a repository from the Internet"
  4. Select your "schoolhub-platform" repository and choose a local path
  5. Click "Clone" to download the empty repository
  6. Copy all files from your extracted CodeCanyon folder to the cloned repository folder
  7. In GitHub Desktop, you'll see all files as "new files"
  8. Add a commit message: "Initial commit"
  9. Click "Commit to main" then "Push origin"

Step 6: Verify Upload

  1. Go to your GitHub repository page
  2. Verify all files are uploaded (package.json, app/, components/, etc.)

Next Steps

Now that your code is on GitHub, you can proceed with deployment:

  • Deploy to Vercel (Recommended for beginners)
  • Deploy to AWS EC2 (For advanced users)
Security Note: Never commit actual API keys to GitHub. Keep sensitive information in .env.local files.
